Configuração Inicial no Figma

01

Capa e Nomeação do Arquivo 💡

Nomeação: Siga o padrão do time para que o arquivo seja facilmente localizável no Jira e no Figma.

Padrão sugerido
[Código do Jira] [Nome do Projeto] - Squad [Nome da Squad]
Ex: OL-404 - Checkout Simplificado - Squad Logística

Capa (Thumbnail): Todo arquivo criado do zero deve ter uma capa na primeira página utilizando o componente oficial de Capa do Design System, contendo o status do projeto, nome e designer responsável.

Em progresso Em Review Pronto para Handoff
02

Vinculando as Bibliotecas Corretas

No menu de Libraries do arquivo, ative as bibliotecas corretas para evitar componentes duplicados ou cores erradas:

Foundation Cores, tipografia, elevações e tokens de espaçamento
Web Components Botões, inputs, cards e elementos de interface
Assets Ícones, ilustrações e logos oficiais
03

Estrutura de Páginas do Arquivo

Sempre separe o arquivo em páginas diferentes para melhorar a performance e facilitar a navegação de stakeholders e engenheiros.

📌 Capa Apenas o card do projeto
🔍 Discovery & Research Benchmarking, fluxos antigos, prints, anotações e hipóteses
🎨 Playground / Rascunhos Onde você é livre para criar, errar e testar ideias
🧪 Protótipos / Testes Fluxos interativos montados para testes com usuários
🚀 Ready for Dev / Handoff Telas validadas e finais. Nunca altere sem avisar o squad.
🗄️ Arquivo / Trash Telas descartadas. Não delete — guarde para histórico.
04

Aplicação de Temas e Modos

A Overlabs atende diferentes marcas dentro da holding Waiken ILW. Configure os temas no painel de propriedades do Figma (canto superior direito):

1. Selecione a seção ou página inteira que vai trabalhar.

2. No painel de variáveis, selecione o tema de nível superior: company-type.

3. Filtre para o subtema específico.

Exemplo prático
Mude o modo para company-type → sky-products. Dentro de sky-products, configure para sky dark. Os componentes e cores se transformarão automaticamente.
02

Padronização de Design

📐

Grade e Espaçamentos

  • 8pt Grid: todos os espaçamentos seguem a regra do grid de 8 pontos (8px, 16px, 24px, 32px, 48px...)
  • Para ajustes internos densos, utilize o subgrid de 4px
  • Auto Layout obrigatório em todas as estruturas — listas, cards, formulários e páginas
🏷️

Higiene de Camadas

  • Sem Detach — nunca quebre componentes do Design System. Precisa de variação? Fale com o time de Design System
  • Renomeie camadas — sem nomes genéricos. Use Frames com nomes claros: Header, Card - Product Info, Button / Primary
🗺️

Fluxos e Documentação

  • Use Sections (Shift+S) — agrupe telas do mesmo fluxo com título claro
  • Fluxos da esquerda para direita — simule a jornada do usuário linearmente
  • Notas para Devs — documente regras de negócio com caixas de texto ou stickers coloridos
⚠️

Nunca dê detach em componentes do Design System. Se precisar de uma variação que não existe, fale imediatamente com o time de Design System no Microsoft Teams para criarmos juntos. Manter a consistência é responsabilidade de todos.
